# Template de tutorial

Copie o modelo abaixo sempre que for criar um novo tutorial para o Maxflow.

```md
# [Titulo da funcionalidade]

## Quando usar
Use esta tela quando [explicar em que situacao a funcionalidade deve ser usada].

Exemplo prático:
[descrever um caso real do dia a dia]

## O que essa tela faz
Esta tela permite [explicar de forma simples o objetivo da funcionalidade].

## Passo a passo
1. Acesse o menu **[nome do menu]**.
2. Clique em **[nome da opcao]**.
3. Clique em **[acao principal]**.
4. Preencha os campos necessarios.
5. Revise as informacoes.
6. Clique em **Salvar** ou **Confirmar**.

Exemplo prático:
Se voce precisa [situacao real], faca este processo para [resultado esperado].

## Explicacao dos campos
- **[Campo 1]**: [explicacao simples e objetiva].
- **[Campo 2]**: [explicacao simples e objetiva].
- **[Campo 3]**: [explicacao simples e objetiva].

## Dicas importantes
- [dica pratica 1]
- [dica pratica 2]
- [dica pratica 3]

## Erros comuns
- **[Erro 1]**: [como identificar e corrigir].
- **[Erro 2]**: [como identificar e corrigir].
- **[Erro 3]**: [como identificar e corrigir].

## Perguntas frequentes
### [Pergunta frequente 1]
[Resposta simples e direta]

### [Pergunta frequente 2]
[Resposta simples e direta]

### [Pergunta frequente 3]
[Resposta simples e direta]
```

## Boas praticas

- usar titulos claros
- manter frases curtas
- explicar apenas o que o usuario precisa para concluir a tarefa
- evitar termos internos do desenvolvimento
- usar exemplos práticos sempre que possivel

## Checklist antes de publicar

- o tutorial explica quando usar a tela
- o passo a passo esta em ordem
- os campos importantes foram explicados
- existem exemplos práticos
- a linguagem esta simples
- nao foram inventadas funcionalidades
