# Visão geral do Maxflow

O Maxflow é um ERP voltado para empresas prestadoras de serviço. A central de ajuda deve acompanhar a jornada real de quem cadastra clientes, monta orçamentos, executa serviços e acompanha o financeiro.

## Público principal

- oficinas
- prestadores de serviço
- empresas de serviços
- usuários com pouca familiaridade técnica

## Categorias principais

- Comece aqui
- Cadastros
- Operação
- Fiscal
- Financeiro
- Gestão
- Configurações
- FAQ
- Suporte

## O que cada usuário precisa encontrar rápido

- como fazer uma tarefa
- onde clicar
- o que revisar se der erro
- como pedir ajuda

## Exemplo de imagem no projeto

As imagens podem ficar dentro do próprio projeto e ser vinculadas nos arquivos de documentação.

Estrutura usada neste exemplo:

```text
static/
  img/
    exemplos/
      tutorial-ordem-servico-exemplo.svg
```

Exemplo renderizado na página:

![Exemplo de imagem de tutorial do Maxflow](/img/exemplos/tutorial-ordem-servico-exemplo.svg)

Legenda do exemplo:
- a imagem está dentro do projeto
- o arquivo foi salvo em `static/img/exemplos`
- a documentação acessa a imagem usando o caminho `/img/exemplos/nome-do-arquivo.svg`

## Como usar esse padrão nos próximos tutoriais

1. Salve a imagem em `static/img/exemplos` ou em outra pasta organizada dentro de `static/img`.
2. No arquivo `.md` ou `.mdx`, use o caminho começando com `/img/`.
3. Exemplo:

```md
![Tela de Ordem de Serviço](/img/exemplos/tutorial-ordem-servico-exemplo.svg)
```
