# Perfis de Usuários

## Quando utilizar

Use esta funcionalidade quando a empresa precisar organizar o que cada colaborador pode acessar dentro do Maxflow.

Exemplos reais:

- criar um perfil para a equipe de atendimento
- montar um perfil para o financeiro
- deixar a gerência com acesso mais amplo
- criar um perfil apenas para consulta
- padronizar o acesso de várias pessoas que fazem o mesmo trabalho

## Como acessar

1. No menu do sistema, acesse **Configurações**.
2. Entre em **Perfis de Usuários**.
3. Na listagem, escolha a ação desejada:
   - **Cadastrar**
   - **Alterar**
   - **Detalhes**
   - **Excluir**
   - **Módulos** para visualizar o que o perfil já possui

## Passo a passo

### Como criar um perfil

1. Acesse a tela de **Perfis de Usuários**.
2. Clique em **Cadastrar**.
3. Preencha o campo **Nome**.
4. Vá para a área de **Módulos**.
5. Se quiser, use o campo **Menu** para filtrar os módulos por grupo.
6. Deixe marcados apenas os módulos que esse perfil deve acessar.
7. Em cada módulo, marque as ações que esse perfil poderá fazer.
8. Revise com atenção.
9. Clique em **Salvar**.

Exemplo real:

- uma empresa cria o perfil **Atendimento** e libera acesso a **Pessoas**, **Orçamentos** e **Ordens de Serviço**, sem liberar acesso ao financeiro.

### Como alterar um perfil

1. Na listagem, localize o perfil.
2. Clique em **Alterar**.
3. Ajuste o **Nome**, os **Módulos** ou as **Permissões**.
4. Revise as alterações.
5. Clique em **Salvar**.

Exemplo real:

- a equipe financeira passou a cuidar também do contas a pagar. O usuário abre o perfil **Financeiro**, libera esse módulo e salva novamente.

### Como visualizar os módulos de um perfil

1. Na listagem, localize o perfil desejado.
2. Clique no botão que mostra a quantidade de módulos.
3. Revise os módulos e as permissões exibidas no modal.

Exemplo real:

- antes de vincular um perfil a um novo colaborador, o gestor abre a visualização rápida para conferir se o perfil está correto.

### Como ver os detalhes do perfil

1. Na listagem, clique em **Detalhes**.
2. Confira as informações do perfil:
   - identificador
   - nome
   - data de cadastro
   - data da última alteração
   - módulos liberados
   - permissões

### Como excluir um perfil

1. Na listagem, clique em **Excluir**.
2. Confirme a exclusão.
3. Aguarde a resposta do sistema.

Importante:

- se o perfil já estiver vinculado a colaboradores, a exclusão pode ser bloqueada

## Explicação dos campos principais

- **Nome**: identifica o perfil que será usado no cadastro dos colaboradores.
- **Menu**: ajuda a filtrar os módulos por grupo, deixando a escolha mais organizada.
- **Módulo**: representa uma área do sistema que poderá ser liberada para o perfil.
- **Permissões**: mostram o que esse perfil poderá fazer dentro de cada módulo.
- **Identificador**: código interno do perfil, exibido na tela de detalhes.
- **Data de cadastro**: mostra quando o perfil foi criado.
- **Data da última alteração**: mostra a última vez em que o perfil foi alterado.
- **Funções**: lista de funções vinculadas ao perfil na tela de detalhes.

Exemplos simples de permissões:

- **Listar**: ver a tela
- **Cadastrar**: criar novos registros
- **Alterar**: editar registros existentes
- **Excluir**: remover registros
- **Detalhar**: abrir a visualização completa

## Situações práticas de uso

- **Perfil de atendimento**: para quem precisa trabalhar com clientes, orçamentos e ordens de serviço.
- **Perfil financeiro**: para quem precisa cuidar de contas a pagar, contas a receber e relatórios financeiros.
- **Perfil da gerência**: para quem precisa enxergar mais áreas do sistema.
- **Perfil de consulta**: para quem só precisa visualizar informações, sem alterar dados.
- **Perfil operacional**: para quem trabalha no dia a dia da operação e precisa de acesso só às rotinas da equipe.

## Dicas importantes

- Use nomes claros, como `Atendimento`, `Financeiro`, `Gerência` ou `Consulta`.
- Revise todos os módulos antes de salvar.
- Na criação, a tela pode trazer módulos e permissões já marcados. Por isso, vale revisar com atenção antes de confirmar.
- Libere apenas o que a equipe realmente precisa usar.
- Se várias pessoas fazem a mesma função, use o mesmo perfil para manter o padrão.
- Antes de vincular um perfil a um colaborador, abra a visualização de **Módulos** para conferir o acesso.
- Se o sistema mostrar uma mensagem para evoluir o plano ao tentar mexer nas permissões, isso indica limitação do plano contratado para esse recurso.

## Erros comuns

- **Salvar sem informar o nome**: o nome do perfil é obrigatório.
- **Criar dois perfis com o mesmo nome**: o sistema não permite nome repetido.
- **Deixar o perfil sem módulos**: o perfil precisa ter pelo menos um módulo liberado.
- **Liberar módulos demais**: isso pode dar acesso a áreas que o colaborador não precisa usar.
- **Não revisar as permissões**: um módulo pode ficar com acesso maior do que o necessário.
- **Tentar excluir um perfil em uso**: o sistema pode bloquear quando existem colaboradores vinculados.
- **Usar nomes genéricos demais**: depois fica difícil saber qual perfil escolher.

## Perguntas frequentes

### Posso criar dois perfis com o mesmo nome?

Não. O nome do perfil precisa ser único.

### Um perfil pode ficar sem nenhum módulo?

Não. O sistema exige pelo menos um módulo liberado.

### Posso alterar um perfil depois de criado?

Sim. O perfil pode ser alterado sempre que necessário.

### Posso ver o que um perfil faz antes de usar?

Sim. Você pode usar o botão de **Módulos** na listagem ou abrir a tela de **Detalhes**.

### Posso excluir um perfil que já está em uso?

Normalmente não. Se houver colaboradores vinculados a esse perfil, o sistema pode bloquear a exclusão.

### O sistema mostra status do perfil, como ativo ou inativo?

Na análise feita, não foi encontrado um status de ativo ou inativo para perfis de usuários.

### O que acontece quando todas as permissões de um módulo são liberadas?

Na prática, o perfil fica com acesso completo àquele módulo.

### Por que apareceu uma mensagem para evoluir o plano?

Porque o controle mais detalhado de permissões depende do recurso de controle de acesso disponível no plano da empresa.

### O botão Histórico aparece para todos os usuários?

Não necessariamente. Na análise do sistema, esse botão aparece em condição especial de acesso.
